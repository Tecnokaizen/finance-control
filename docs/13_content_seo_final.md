# 13 · Content SEO

## 1. Propósito del documento

Este documento define la estrategia de contenido SEO para la app de control financiero. Su objetivo es estructurar la capa pública del producto, decidir qué páginas conviene crear, qué intenciones de búsqueda atacar y cómo alinear el contenido con las funcionalidades reales del MVP.

Debe servir como referencia para:

- arquitectura SEO del producto,
- páginas públicas principales,
- landings por funcionalidad,
- clusters de contenido,
- copy base para captación,
- metadatos y semántica,
- roadmap editorial.

---

## 2. Objetivo del SEO del producto

La estrategia SEO debe ayudar a:

- captar tráfico orgánico cualificado,
- explicar claramente el valor del producto,
- posicionar funcionalidades clave,
- conectar búsquedas informacionales con páginas de conversión,
- construir una base escalable para marketing futuro.

---

## 3. Enfoque recomendado

### Recomendación principal
Conviene separar dos capas:

### 3.1 Capa producto / conversión
Páginas orientadas a convertir:
- home del producto,
- páginas de funcionalidades,
- páginas de casos de uso,
- pricing o acceso si más adelante aplica,
- página de contacto o demo si existe.

### 3.2 Capa contenido / blog / recursos
Páginas orientadas a captar demanda informacional:
- guías,
- comparativas,
- tutoriales,
- artículos evergreen,
- contenidos educativos relacionados con control financiero.

### Regla
El SEO no debe prometer funcionalidades que el MVP aún no tiene.

---

## 4. Estado recomendado del proyecto a nivel SEO

### Fase actual recomendada
- app privada funcional primero,
- capa pública ligera pero bien estructurada,
- contenido SEO incremental después del núcleo.

### Implicación práctica
No hace falta lanzar un portal enorme desde el día uno. Basta con:
- una home clara,
- 3 a 5 landings de funcionalidad,
- una base inicial de artículos de alta intención.

## 14. Estructura recomendada de una landing funcional

### Bloques sugeridos
1. Hero con propuesta clara.
2. Problema que resuelve.
3. Cómo funciona.
4. Beneficios principales.
5. Capturas o explicación visual.
6. Casos de uso.
7. FAQs.
8. CTA final.

### Ejemplo de hero
- título centrado en resultado,
- subtítulo centrado en claridad y ahorro de tiempo,
- CTA principal,
- apoyo visual del dashboard o flujo.

## 15. Estructura recomendada de un artículo SEO

### Bloque tipo
1. Introducción breve.
2. Definición o contexto.
3. Problema habitual.
4. Solución paso a paso.
5. Errores comunes.
6. Recomendación práctica.
7. CTA o enlace a funcionalidad relacionada.

### Reglas de estilo
- claro,
- útil,
- orientado a intención real,
- sin relleno,
- con ejemplos si procede.

## 16. Metadatos y on-page SEO

### Recomendaciones básicas
- un H1 claro por página,

- meta description persuasiva y realista,
- URLs limpias y cortas,

## 18. Riesgos SEO a evitar

### Riesgos comunes
- atacar keywords demasiado grandes desde el inicio,
- crear muchas páginas finas sin profundidad,
- prometer OCR o automatización que el MVP no tiene,
- mezclar lenguaje de contabilidad avanzada si el producto no llega a ese nivel,
- canibalizar páginas de funcionalidad con artículos demasiado parecidos.

### Mitigación
- pocas landings, bien diferenciadas,
- cluster por intención,
- copy honesto sobre lo que hace el producto,
- interlinking claro.

## 22. Recomendaciones para Codex

Cuando Codex trabaje la capa pública o el marketing site debe:

- respetar el alcance real del MVP,
- no prometer integraciones o automatizaciones no implementadas,
- estructurar páginas pensando en intención de búsqueda y claridad,
- reutilizar el lenguaje definido en producto y funcionalidades,
- conectar cada landing con un CTA claro,
- mantener URLs y headings limpios.
